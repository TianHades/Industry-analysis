# 竞争格局分析 — 成熟期的盈利性核心

## R — 原文 (Reading)

> 成熟期的核心问题是"谁能胜出？怎么分蛋糕？"——横向看集中度（CRn），纵向看产业链利润分配。行业集中度随产业生命周期的演变先降后升——导入期少数先行者集中度高→成长期大量玩家涌入集中度下降→成熟期优胜劣汰集中度回升。
>
> — 肖璟, 第5章

---

## I — 方法论骨架 (Interpretation)

### Know-What：集中度指标

- **CR3 / CR8**：前N名企业销售额占比
- 优先计算CR8（行业标准），同时给出CR3便于快速判断
- **⚠️ CRn 数据质量警示**：Wind/Choice 等终端统计的 CRn 只含上市公司数据。若行业上市公司很少，CR8 可能虚高至 90%+ 完全失真。更合理的做法是：将上市公司销售额加总 ÷ 全行业测算市场规模

### 市场结构分类与资本市场态度

| 结构类型 | 细分 | CR8 范围 | 资本市场态度 |
|---------|------|---------|------------|
| **竞争型** | 分散竞争型 | 0% < CR8 < 20% | 看重成长空间大的企业 |
| | 低集中竞争型 | 20% ≤ CR8 < 40% | 看重成长空间大的企业 |
| **寡占型** | 低集中寡占型 | 40% ≤ CR8 < 70% | 看重能持续扩大份额的龙头 |
| | 极高寡占型 | 70% ≤ CR8 ≤ 100% | 寡头估值差异不大，看重稳定性 |

### Know-Why：集中度 U 型演变

```
导入期 → 少数先行者，集中度高
成长期 → 大量玩家涌入，集中度下降
成熟期 → 优胜劣汰，龙头兼并，集中度回升
衰退期 → 弱者出清，集中度继续上升
```

### 集中度变化的驱动因素

| 驱动因素 | 影响方向 | 示例 |
|---------|---------|------|
| 宏观经济下行 | 集中度 ↑ | 弱者出清，龙头抗风险强 |
| 宏观经济上行 | 集中度 ↓ | 中小企业生存空间改善 |
| 刺激需求政策 | 集中度 ↓ | 不具备竞争力的厂商存活 |
| 限制供给政策 | 集中度 ↑ | 落后产能加速出清 |
| 价格战/兼并重组 | 集中度 ↑ | 存量竞争促进行业整合 |

**投资应用**：不仅要回答"CRn 现在是多少"，更要回答"CRn 正在朝哪个方向走、为什么"。

### 纵向分析：产业链利润分配

产业链各环节的利润水平差异往往比同一环节内企业间的差异更大——这是判断行业投资价值的重要维度：

| 环节 | 毛利率水平 | 议价力分析 |
|------|-----------|-----------|
| 上游 | {数据}（来源） | vs 中游谁更强？ |
| 中游 | {数据}（来源） | 受两端挤压还是能转嫁？ |
| 下游 | {数据}（来源） | 品牌溢价还是成本敏感？ |

## A1 — 书中的应用 (Past Application)

### 案例 1: 空调行业的集中度U型演变
- **问题**: 空调行业从1990年代导入期到2010年代成熟期，集中度如何演变？
- **方法论的使用**: 导入期（1990-1996，渗透率<5%）→ 成长期（1997-2005，渗透率16%→81%）→ 价格战洗牌（2001-2004，CR3从37%升至55%）→ 成熟期（2005+，双寡头格局稳定）
- **结论**: U型演变得到完整验证——导入期集中度高（格力春兰少数玩家），成长期下滑（大量品牌涌入），成熟期回升（价格战淘汰弱者）
- **结果**: 格力和美的在价格战后成为赢家，长期投资者获得超额回报

### 案例 2: 餐饮住宿行业的CRn失真警示
- **问题**: 如果用Wind终端统计餐饮住宿行业的CR8，结果可信吗？
- **方法论的使用**: 餐饮住宿行业上市公司极少（海底捞、华住等少数），Wind统计的CR8可能虚高至90%+。但实际全国餐饮市场极度分散，连锁化率才约20%
- **结论**: CRn数据必须核实覆盖范围——只含上市公司的CRn在上市率低的行业中严重失真
- **结果**: ✅ 标注数据局限性，改用"上市企业销售额 ÷ 全行业测算市场规模"替代

## A2 — 触发场景 (Future Trigger) ★

### 语言信号

- "行业集中度怎么样？"
- "CR3/CR5/CR8是多少？"
- "谁是这个行业的龙头？"
- "这个行业会不会越来越集中？"
- "为什么有些行业很分散、有些很集中？"
- "前几名加起来占多少份额？"

### 与相邻 skill 的区分

- 与 `moat-analysis` 的区别: 护城河看"**单家**企业为何能守住利润"（防守性），竞争格局看"**行业层面**的利润如何分配"（盈利性）。两者构成成熟期二维分析矩阵。
- 与 `capacity-cycle-judgment` 的区别: 产能周期是集中度变化的深层驱动力之一——产能出清期集中度加速上升。

## E — 可执行步骤 (Execution)

1. **计算横向集中度**
   - 搜索CR3/CR8数据，优先最近的行业报告
   - 搜索执行参考: `reference/search-specs.md` 的成熟期搜索指引
   - 数据标注参考: `reference/data-annotation-standards.md`
   - 核实CRn覆盖范围——是否只含上市公司？是否需要手动修正？
   - 完成标准: CR3 + CR8 + 市场结构类型判断 + 数据质量说明

2. **判断集中度变化方向**
   - 对比历史CRn趋势（至少回溯3年）
   - 识别当前驱动因素（宏观政策/技术替代/兼并重组/价格战）
   - 完成标准: "CRn正在向XX方向走，因为XX"的判断

3. **分析纵向产业链利润分配**
   - 收集上中下游各环节的毛利率/净利率
   - 判断利润在哪个环节最集中，哪个环节被挤压
   - 完成标准: 产业链利润分布表

4. **TOP企业对比**
   - 列出TOP5-10企业，比较市场份额、核心优势、最新动态
   - 完成标准: 企业对比表，含市占率/优势/动态/数据来源

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 导入期行业——竞争格局尚未形成，CRn计算无意义
- 纯企业层面的竞争对手分析——这是公司分析而非行业分析

### 作者在书中警告的失败模式

- **CRn数据失真**：只含上市公司的CRn在上市率低的行业（如餐饮、农业）中严重高估集中度
- **只看静态CRn，不看变化方向**：CR8=40%（低集中寡占型）但如果在快速上升中 → 投资机会大于静态CR8=70%的行业

### 作者的盲点 / 时代局限 (来自 BOOK_OVERVIEW.md)

- **A股中心**：CRn数据主要来自Wind/Choice等A股终端，对非上市企业占比高的行业覆盖不足
