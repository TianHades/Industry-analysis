# 身份导向输出 — 根据受众定制分析方案

## R — 原文 (Reading)

> "解题归解题，沟通归沟通"——研究的推演顺序（提出假设→验证→结论）不等于向读者呈现的顺序。
> SCQR = 情景 → 冲突 → 问题 → 解答。好的分析师会根据受众调整叙事结构：
> 路人先制造冲突感，大老板直接给结论。
>
> — 肖璟, 第11章

---

## I — 方法论骨架 (Interpretation)

### 五大身份分析方案

不同身份的核心关注点不同，分析深度和侧重点也不同：

| 身份 | 核心关注点 | 重点模块 | 叙事顺序 |
|------|-----------|---------|----------|
| **投资者** | 投资价值、风险、估值 | 生命周期、估值、景气度、竞争格局、护城河 | R→S→C→Q（结论前置） |
| **创业者** | 市场机会、进入壁垒、商业模式 | 市场规模、护城河、产业链、商业模式画布 | R→S→C→Q（结论前置） |
| **求职者** | 行业前景、薪酬、头部企业 | 发展阶段、竞争格局、主要企业、景气度 | S→C→Q→R（经典SCQR） |
| **学生/研究者** | 行业全貌、研究方法 | 完整框架、数据来源、研究方法 | S→C→Q→R（经典SCQR） |
| **路人/快速了解** | 基本认知、核心事实 | 执行摘要、核心发现、行业定义 | C→Q→S→R（冲突前置） |

### SCQR叙事顺序与身份匹配

| 受众类型 | 对应身份 | 推荐叙事顺序 | 理由 |
|---------|---------|------------|------|
| **路人** | 快速了解者 | C→Q→S→R（冲突前置） | 先吸引眼球，再给背景 |
| **观众** | 学生/研究者 | S→C→Q→R（经典SCQR） | 对方有兴趣，按逻辑展开 |
| **大老板** | 投资者/创业者 | R→S→C→Q（结论前置） | 时间宝贵，直接给结论 |
| **直属领导** | （不适用） | R only（仅给解答） | 已了解背景，只要方案 |

### 模块权重按身份调整

在生命周期决定的基础权重之上，叠加身份偏好：

| 身份 | 追加侧重的模块 | 可降权的模块 |
|------|-------------|------------|
| 投资者 | 估值 +30%、景气度 +20% | 商业模式（如非导入期） |
| 创业者 | 商业模式 +30%、市场规模 +20% | 估值（如非融资阶段） |
| 求职者 | 竞争格局（头部企业）+30% | 估值 |
| 学生/研究者 | 全模块均衡 +20%方法论 | — |
| 路人 | 执行摘要 +50%、其余简要 | — |

### 身份确认流程

**在分析开始前必须先确认用户身份**。如果用户没有明确说明，主动询问：

> "在开始分析之前，请问你的身份是？"
> - 投资者（关注投资价值和风险）
> - 创业者（关注市场机会和进入壁垒）
> - 求职者（关注行业前景和头部企业）
> - 学生/研究者（关注行业全貌和研究方法）
> - 路人（只需要快速了解核心事实）

## A1 — 应用示例 (Application Examples)

> **说明**：以下案例为本技能库构造的输出示例，展示不同身份对应的报告结构和行动建议。原著 SCQR 叙事框架为理论基础（第11章），身份定制体系为本技能库扩展。

### 示例 1: 投资者版的行动建议
```markdown
## 🚀 建议下一步行动（投资者）

### 立即可做的事
1. **阅读头部企业年报**：中国移动(600941)、中国电信(601728)
2. **关注政策动向**：工信部官网

### 与agent继续研究
1. **运营商估值模型**：对比三大运营商PE、PB、股息率
2. **5G-A投资回报分析**：CAPEX与收入增长的匹配度

### 需要专业工具
1. **使用Wind/Bloomberg**：获取实时估值数据
2. **DCF估值模型**：对云业务分部进行独立估值
```

### 示例 2: 创业者版的行动建议
```markdown
## 🚀 建议下一步行动（创业者）

### 立即可做的事
1. **分析产业链缺口**：上游、中游、下游哪些环节有空白
2. **研究龙头企业的薄弱环节**：服务痛点

### 与agent继续研究
1. **细分赛道选择**：AI+、边缘计算等方向分析
2. **进入壁垒评估**：资金需求、牌照要求、技术门槛

### 需要实地调研
1. **走访目标客户**：了解真实需求和付费意愿
2. **参加行业展会**：了解技术趋势和竞争态势
```

### 示例 3: 求职者版的行动建议
```markdown
## 🚀 建议下一步行动（求职者）

### 立即可做的事
1. **关注头部企业招聘**：公司官网、招聘平台
2. **了解岗位要求**：技术岗、产品岗、销售岗的技能要求

### 与agent继续研究
1. **企业对比分析**：薪酬、福利、发展空间
2. **新兴岗位研究**：AI算法工程师、解决方案架构师等

### 需要人脉网络
1. **联系在职员工**：了解真实工作环境
2. **参加行业沙龙**：拓展人脉，获取内推机会
```

## E — 可执行步骤 (Execution)

1. **确认用户身份**（分析开始前）
   - 如果用户未说明身份，主动询问
   - 完成标准: 明确的身份类型 + 用户确认

2. **生成身份定制方案**
   - 根据身份表确定：重点模块、模块权重调整、叙事顺序、输出深度
   - 完成标准: 一份"该身份的分析方案"——哪些模块要深度、哪些简要

3. **选择叙事顺序**
   - 根据 SCQR 四类受众表，确定报告的叙事结构
   - 投资者/创业者：结论前置（先给核心判断）
   - 学生/研究者：经典逻辑展开
   - 路人：冲突前置（先制造关注点）
   - 完成标准: 报告开头/摘要的叙事顺序已确定

4. **生成身份定制的行动建议**
   - 在报告末尾为该身份生成"下一步建议"
   - 分为三层：立即可做 / 与agent继续研究 / 需人工或专业工具
   - 完成标准: 三个层级的行动建议完整

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 用户只需要一个不涉及决策的纯学术分析——身份定制的前提是"分析要服务于某个决策场景"
- 用户已经明确指定了分析范围和深度——不需要再次做身份定制
- 用户说"随便分析一下"——需要追问一个基本身份方向，不能在没有身份的情况下开始分析

### 容易混淆的情况

- **同一个人可能有多个身份**：如"我既是投资者也是从业者"——请用户选择最主要的视角。"如果只能从一个角度出发，你最关心什么？"
- **身份≠分析能力**：学生身份不意味着分析应该更浅——而是意味着侧重点不同（更关注方法论和完整框架）
