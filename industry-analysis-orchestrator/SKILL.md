# 行业分析总成 — 投资者视角的系统化行业研究

## R — 原文 (Reading)

> 市面上大部分的行业研究课程或书籍，都会列出商业模式、产业链、市场空间、竞争格局等维度的内容……但这么多东西"砸"过来，彼此间又相互割裂。本书引入一个以产业生命周期为核心的行业研究框架——在产业生命周期的不同阶段，行业有着不同的鲜明特征，研究的重点自然也应有所区别。
>
> — 肖璟, 第1章

---

## I — 方法论骨架 (Interpretation)

你是行业分析的总装车间。你的职责不是做某一个具体分析，而是**按正确顺序把 13 个子技能串成一条完整的行业研究流水线**。

核心逻辑：**先定位，再分析，后验证**。

1. **身份确认**：先确定用户身份（投资者/创业者/求职者/学生/路人），不同身份对应不同的模块权重和叙事顺序
2. **定位**：用渗透率判断行业在产业生命周期的哪个阶段——这一步决定后续所有分析维度的权重
3. **定向分析**：根据阶段分配火力——
   - 导入期：80%精力在商业模式可行性 + TAM天花板
   - 成长期：80%精力在市场规模测算 + 渗透率追踪
   - 成熟期：60%精力在护城河+竞争格局，30%在产能周期
   - 衰退期：重点在替代品威胁和第二曲线可能性
4. **估值映射**：行业值多少钱取决于它在哪个阶段
5. **外部修正**：PEST四维扫描修正所有基本面假设
6. **高频验证**：景气度指标体系持续追踪，发现偏差及时修正

**核心原则**：
1. **身份导向**：根据用户身份调整分析深度和侧重点
2. **全面搜索**：不吝啬搜索次数和LLM调用，每个可用渠道都应尝试，确保数据全面、互相验证。**具体搜索方法、关键词模板、工具链见 `reference/search-specs.md`**
3. **信源第一**：所有数据标注来源。**标注格式、时效校验规则、冲突仲裁规则见 `reference/data-annotation-standards.md`**
4. **诚实透明**：明确标注AI能力边界，不确定内容用"🔍不确定"标注
5. **行动导向**：报告结尾给出针对身份的"下一步建议"
6. **输出到文件**：分析完成后，将完整报告写入 `output/{行业名称}-行业分析报告（{身份}版）.md`
7. **时效校验**：所有数据必须通过 `reference/data-annotation-standards.md` 中的时效校验规则——过期数据直接丢弃，不得进入报告
8. **提炼而非总结**：关键结论必须包含分析增量（"那又怎样？"），不只是复述数据。区分事实判断与价值判断
9. **产业链不跳过**：无论行业处于哪个生命周期阶段，产业链结构必须作为独立章节在最终报告中呈现——这是理解一个不熟悉行业的"地图"

## A1 — 书中的应用 (Past Application)

### 案例 1: 新能源汽车行业完整分析
- **问题**: 2020年前后，新能源汽车赛道火热但鱼龙混杂——到底该不该投、投哪个环节？
- **方法论的使用**: 先用渗透率定位（2020年国内新能源车渗透率约5%，处于导入期末端→成长期前端）→ 验证商业模式可行性（对标燃油车需求，真实存在；电池成本快速下降使UE趋向盈亏平衡）→ 测算市场规模（TAM对标全球汽车市场约3万亿美元）→ 分析护城河（电池技术+充电网络+品牌）→ 纵向拆解产业链利润分配（上游锂矿→中游电池→下游整车）→ PEST外部扫描（补贴退坡风险+双碳政策驱动+技术路线不确定性）→ 景气度追踪（月度销量、电池排产、锂价）
- **结论**: 行业处于成长期前端，投资重点应在确定性最高的中游电池环节
- **结果**: 2020-2022年新能源汽车板块迎来戴维斯双击，但2023年价格战后整车环节利润严重分化

### 案例 2: 空调行业全生命周期复盘
- **问题**: 空调行业从1990年代导入期到2010年代成熟期，完整走过产业生命周期
- **方法论的使用**: 导入期（1990-1996，渗透率<5%）→ 成长期（1997-2005，渗透率16%→81%）→ 价格战洗牌（2001-2004，CR3从37%升至55%）→ 成熟期（2005+，双寡头格局稳定）
- **结论**: 估值完美跟踪产业生命周期：导入期0.4倍→成长期0.8倍→价格战中估值回落→格局改善后重回0.8倍
- **结果**: 格力和美的在价格战后成为赢家，长期投资者获得超额回报

## E — 可执行步骤 (Execution)

> **搜索执行规范**：以下各阶段涉及数据搜索时，**必须遵循 `reference/search-specs.md`** ——使用三步工具链（Chrome搜索→extract提取→WebFetch深度阅读）、按生命周期定向搜索关键词、执行搜索原则（渠道穷尽/不设上限/并行搜索/反面观点搜索）。
>
> **数据质量标准**：所有收集到的数据**必须通过 `reference/data-annotation-standards.md` 中的时效校验规则**，过期数据直接丢弃。数据冲突时按仲裁规则采信。

### 第一阶段：身份确认 + 定位（5分钟）

**1. 确认用户身份** → 调用 `identity-oriented-output`
   - 询问用户身份（如未说明）
   - 根据身份确定模块权重、叙事顺序、输出深度
   - 完成标准: 已确认身份类型 + 已输出身份定制方案

**2. 行业定义与边界** → 调用 `industry-definition`
   - 确定行业定义、分类归属、边界
   - 判断是否需要拆分为子赛道
   - 绘制产业链结构图
   - 完成标准: 明确的分析范围 + 产业链结构

**3. 定位产业生命周期阶段** → 调用 `industry-lifecycle-anchoring`
   - 估算渗透率 → 判断阶段 → 评估四种偏离可能
   - 输出该阶段的分析重点矩阵（★★★/★★☆/★☆☆）
   - 完成标准: 明确阶段判断 + 渗透率依据 + 优先级矩阵
   - 判停条件: 无法估算渗透率 → 用辅助信号推断

**4. 建立初始假设** → 调用 `hypothesis-driven-research`
   - 基于阶段判断，提出3-5个可验证的关于该行业的核心假设
   - 完成标准: 假设清单，每项可被后续分析验证或证伪

### 第二阶段：基本面深度分析（15-20分钟）

**5. 时空对标建立参照系** → 调用 `spatial-temporal-benchmarking`
   - 时间对标：历史上类似需求如何被满足？
   - 空间对标：成熟市场对应行业的关键指标？
   - 终局对标：稳态下的市场规模/集中度/利润率？
   - 完成标准: 至少1个时间对标 + 1个空间对标市场

**6. 按生命周期阶段分配分析深度**：

   **导入期** → 调用 `business-model-feasibility`（★60%）+ `market-size-estimation`（★30%）
   - 销售可行性：需求真实吗？（时间对标+空间对标）
   - 利润可行性：搭建三版本UE模型（当前→盈亏平衡→终局）
   - TAM天花板：潜在市场有多大？
   - 其余模块简要带过

   **成长期** → 调用 `market-size-estimation`（★60%）+ `industry-lifecycle-anchoring` 追踪渗透率（★20%）
   - TAM/SAM/SOM三口径测算（乐观/中性/悲观三情景）
   - 渗透率增速追踪、拐点判断
   - 竞争格局初判
   - 其余模块简要带过

   **成熟期** → 调用 `moat-analysis`（★40%）+ `competitive-landscape-analysis`（★30%）+ `capacity-cycle-judgment`（如有周期化特征，★30%）
   - 护城河：最相关的2-3类深度分析 + 可绕过性检验
   - 竞争格局：CRn趋势 + U型演变判断 + 产业链利润分配
   - 产能周期：CAPEX/折旧比 + 库存水位
   - 其余模块简要带过

   **衰退期** → 重点：替代品威胁 + 第二曲线可能性 + 估值陷阱判断
   - 谁在替代？替代速度？
   - 龙头有无新业务/新市场/新技术？
   - 供给收缩 vs 需求收缩——有无"回光返照"机会？

   > 🔴 **各阶段强制模块**：无论行业处于哪个生命周期阶段，**产业链结构**（`industry-definition` 中的产业链绘制部分）必须在最终报告中作为独立章节呈现（对应输出模板的 `### 10. 产业链结构`）。理由：产业链结构是理解一个不熟悉行业的"地图"——读者可能不了解这个行业，但看过产业链图后就能定位各环节的参与者、利润分配和投资机会。这不随生命周期阶段变化而改变其基础价值。

### 第三阶段：外部修正与验证（10分钟）

**7. 估值判断** → 调用 `lifecycle-valuation-map`
   - 根据阶段确定估值框架
   - 收集估值数据：优先使用 AkShare（`pip install akshare -q`；用法见 `lifecycle-valuation-map` 子技能 E-执行步骤）获取 PE/PB/股息率/总市值；备用方案：WebFetch 抓取东方财富/雪球行情页
   - 🔴 **投资者版强制要求**：至少 5 家核心标的的估值数据必须为实时数值（非估算区间），数据获取时间标注到日。如 API 和 WebFetch 均失败，在估值章节开头用 🔴 显著标注"以下为 AI 估算区间，未经实时数据验证"
   - 给出估值水平判断（偏低/合理/偏高）+ 跨行业估值对比
   - 完成标准: 估值方法选择 + 投资者版≥5家实时估值数据（其他版≥3家） + 偏低/合理/偏高判断 + 预期差识别

**8. PEST外部驱动力扫描** → 调用 `external-force-scanner`
   - P(政治)：产业政策、国际环境。**优先搜索政府/发改委/部委官网获取政策文件编号**（如"发改价格〔2026〕114号"），涉及重大政策时标注文件编号而非仅描述"政策支持"
   - E(经济)：板块周期敏感度、货币三价格影响路径
   - S(社会)：人口结构、文化变迁
   - T(技术)：颠覆性威胁、Gartner曲线位置、创新层级
   - **关键**：将扫描结果回馈到步骤6——基本面判断需要修正吗？
   - 完成标准: 每个PEST维度至少1个关键驱动/风险因素

**9. 景气度追踪方案** → 调用 `prosperity-tracking-system`
   - 筛选3-5个核心高频指标
   - 设定"验证/证伪"条件清单
   - 完成标准: 追踪方案表 + 验证条件清单

> 🔗 **历史对照（在 StockReview 工程完成）**：本工程职责到产出含完整基本面数据的报告为止。"像→不像→像"历史对照由 StockReview 原生技能 `historical-stage-benchmarking` 承载。报告产出后，按 `reference/如何接入StockReview做历史对照.md` 的 14 维特征向量映射表交接契约，切到 StockReview 执行。

### 第四阶段：综合报告输出（5-10分钟）

**10. 质量自检**
   - 对照 `reference/data-annotation-standards.md` 中的**报告质量自检清单**逐项检查：
     - [ ] 每个数据都有来源标注和时间标签
     - [ ] 区分了事实判断和价值判断
     - [ ] 搜索了反面观点和信息来源
     - [ ] 各模块结论之间无矛盾
     - [ ] 🔒和🔍标注合理，未过度承诺 AI 能力
     - [ ] 关键结论是"提炼"而非"总结"——包含了分析增量，不只是复述数据
     - [ ] 产业链结构已作为独立章节呈现（各阶段强制）
   - 🔴 **投资者版专项加检**：
     - [ ] 估值表中≥5家标的，每家有具体PE/PB/市值数值（非估算区间），标注数据获取时间
     - [ ] 至少引用1个行业指数/板块涨跌幅信号（如"中证绿色电力指数年初至今+20%"）
     - [ ] 涉及行业重大政策时标注了文件编号（如"发改价格〔2026〕114号"）
     - [ ] 核心结论可用一句话概括（"电梯演讲"测试——读完执行摘要就应知道最该做什么）

**11. 生成结构化行业分析报告**
   - 使用 `reference/output-template.md` 作为报告模板
   - 按SCQR叙事顺序排列章节（投资者/创业者=结论前置R→S→C→Q，学生=逻辑展开S→C→Q→R，路人=冲突前置C→Q→S→R）
   - 所有数据遵循 `reference/data-annotation-standards.md` 的标注标准
   - 区分事实判断和价值判断
   - 写入 `output/{行业名称}-行业分析报告（{身份}版）.md`
   - 向用户展示报告摘要（执行摘要+核心结论+下一步建议）

## 🔬 研究元能力速查

> 以下方法论来自原著第10-13章，是高质量行业研究的"元能力"。详细展开见 `hypothesis-driven-research` 子技能。

### 金字塔原理：提炼 > 总结

分析师的核心价值在于通过不断追问"那又怎样？"（So what?）产生深度洞见：

```
罗列资讯 → 寻找规律 → 提炼要点 → 归纳核心
   ↓            ↓           ↓           ↓
 完整语句     分组归类    "说明了什么？"  "那又怎样？"
```

**关键区别**：
- 总结 = "A增长了20%，B下降了5%" → 信息层级，无增量
- 提炼 = "行业呈现结构性分化，龙头增速远超中小玩家" → 知识层级，有洞见

### 假设驱动研究法

不是"先搜数据再填模板"，而是"先提假设再验证"：

```
步骤1: 提出初始假设 → 步骤2: 定向搜索验证 → 步骤3: 修正并得出结论
```

搜索的目的不是"发现数据"，而是"验证或推翻假设"。找到反对证据时立即修正假设方向。

### 信息茧房防范

- 主动搜索**反面观点**（如"{行业} 风险"、"为什么{行业}不行"）
- 使用 Chrome 无痕模式搜索，减少个性化算法干扰
- 同一问题用不同角度提问，交叉比对结果

### 事实判断 vs 价值判断

| 类型 | 定义 | 示例 |
|------|------|------|
| **事实判断** | 可证实或证伪 | "2024年渗透率30%" |
| **价值判断** | 因人而异，无法证实 | "这个行业值得投资" |

> 高质量的价值判断必须有事实判断的支撑。报告中应明确区分两种判断。

### SCQR 叙事框架

研究的推演顺序 ≠ 向读者呈现的顺序。按受众调整：

| 受众 | 对应身份 | 叙事顺序 | 理由 |
|------|---------|---------|------|
| **大老板** | 投资者/创业者 | R→S→C→Q（结论前置） | 时间宝贵，直接给判断 |
| **观众** | 学生/研究者 | S→C→Q→R（经典展开） | 对方有兴趣，按逻辑走 |
| **路人** | 快速了解者 | C→Q→S→R（冲突前置） | 先吸引眼球，再给背景 |

> S=情景 C=冲突 Q=问题 R=解答。又名"空雨湿伞"模型。

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- **单只股票的买卖时机判断**: 本 skill 在行业层面（中观），不涉及个股的财务模型和估值精算
- **宏观经济预测**: 虽然外部扫描涉及宏观因素，但本 skill 不预测GDP/CPI/利率走势——只评估对特定行业的传导路径
- **纯数据查询**: "XX行业去年市场规模多少"是信息检索，不是行业分析
- **极度新兴、无任何参照的行业**: 如果渗透率无法估算、没有时空对标物、没有景气度指标——坦诚告知用户"当前信息不足以做系统分析"
- **短期交易**: 本 skill 分析的是中长期行业趋势（3-5年），不适用于"下周这个板块会涨吗"

### 作者在书中警告的失败模式

- **面面俱到平均用力**: 在每个生命周期阶段都花同样的精力分析所有维度，是最常见的错误
- **渗透率刻舟求剑**: 用存量口径而非增量口径判断长替换周期行业
- **忽略外来者无序扩产信号**: 当行业外玩家开始涌入时，行业顶部通常不远了
- **市盈率滥用**: 对亏损企业（导入期）算PE无意义；对强周期股用PE会买在利润高点
- **静态UE模型陷阱**: 只用当前UE就否定一个行业——必须看三版本UE

### 作者的盲点 / 时代局限 (来自 BOOK_OVERVIEW.md)

- **A股/大行业偏好**: 框架默认投资对象是可上市的大行业，系统性低估小型/利基市场
- **精英信息环境**: 专家访谈、草根调研等方法对普通个人投资者的可操作性有限
- **"快速"的边界**: 本书框架在"一周全职"条件下最佳。如果用户只有2小时，应做简化版——重点只放在生命周期定位+1-2个最相关的维度，而非全流程。**请在分析开始时评估用户的时间预算。**

### 2小时简化版 vs 1周完整版

| 维度 | 2小时简化版 | 1周完整版 |
|------|-----------|----------|
| 阶段判断 | ✅ 必做 | ✅ 完整渗透率分析 |
| 最相关1-2个模块 | ✅ 深度 | ✅ 完整 |
| 其余模块 | 简要带过/跳过 | ✅ 适度展开 |
| 估值 | 简要判断 | ✅ 含历史分位数 |
| PEST | 简要扫描 | ✅ 完整四维 |
| 景气度 | 给出3个指标 | ✅ 完整追踪方案 |
| 搜索深度 | 优先现成报告 | 手动测算+交叉验证 |
