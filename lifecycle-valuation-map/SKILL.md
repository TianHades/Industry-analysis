# 产业生命周期估值地图 — 不同阶段用不同估值逻辑

## R — 原文 (Reading)

> 股票收益 = 盈利增长 + 估值变化。不同生命周期的估值逻辑完全不同——导入期看"梦想"（主题投资），成长期看"超预期"（戴维斯双击），成熟期看"龙头进阶"，衰退期看"供给收缩"。估值由投资者（资金能力+风险偏好）和资产（赔率+概率）共同决定。
>
> — 肖璟, 第6章

---

## I — 方法论骨架 (Interpretation)

### 生命周期-估值对应表

| 阶段 | 渗透率 | 估值方法 | 估值驱动因素 | 典型特征 |
|------|--------|---------|------------|----------|
| **导入期** | < 15-20% | VC估值、主题投资 | 市场空间想象力、技术突破 | 亏损时看"市梦率"，市场愿意为故事买单 |
| **成长期** | 15-40% | PEG、PS | 增速超预期 → 戴维斯双击 | 估值波动大，增速是关键 |
| **成熟期** | > 35-40% | PE、PB、股息率 | 龙头溢价、分红回购 | 估值中枢稳定，关注龙头进阶 |
| **衰退期** | — | 警惕低估值陷阱 | 供给收缩速度 vs 需求下降速度 | PE很低不代表便宜——盈利可能在萎缩 |

### AI 可完成的估值框架分析（不需专业工具）

1. **估值方法匹配**：根据当前生命周期阶段，判断应使用哪种估值方法（VC/PS/PEG/PE/PB），并说明原因
2. **跨行业估值对比**：与相近行业或上下游行业的估值水平对比
3. **历史分位数分析**：若为上市公司，可获取当前 PE/PB 在历史 3-5 年中的分位水平
4. **PEG 简易计算**：若处于成长期，PE ÷ 净利润增速，判断是否 < 1（低估区间）
5. **关键估值驱动因素**：识别影响该行业估值的关键变量（如用户增长、ARPU、政策变化）

### 赔率与概率

| 概念 | 含义 | 对应基本面维度 |
|------|------|--------------|
| **赔率** | 盈利空间与亏损空间的比率（能赚多少） | 规模性 + 盈利性 |
| **概率** | 投资取得正收益的可能性（容不容易赚） | 可行性 + 防守性 |

> **经济周期影响排序**详见 `external-force-scanner` ——不同行业板块对经济周期的敏感度差异巨大，这是估值判断中需要考虑的外部修正因素。

## A1 — 书中的应用 (Past Application)

### 案例 1: 空调行业全生命周期估值复盘
- **问题**: 空调行业从1990年代导入期到2010年代成熟期，估值如何演变？
- **方法论的使用**: 导入期（1990-1996）0.4倍相对估值→成长期（1997-2005）0.8倍→价格战中估值回落→格局改善后重回0.8倍并稳定
- **结论**: 估值完美跟踪产业生命周期——成长期估值最高，成熟期估值趋稳，价格战期估值短暂回落
- **结果**: 在估值回落期（价格战末期、格局改善前夕）入场可获得戴维斯双击收益

### 案例 2: 新能源汽车的估值切换
- **问题**: 新能源汽车行业从导入期到成长期，估值逻辑如何切换？
- **方法论的使用**: 2020年渗透率约5%（导入期）→ 市场给"市梦率"，亏损的车企也能获得高估值 → 2022年渗透率突破20%（成长期）→ 估值逻辑切换为PEG/PS，市场开始关注交付量和毛利率
- **结论**: 渗透率跨越临界点时，估值方法必须同步切换——继续用导入期方式给成长期企业估值会严重高估
- **结果**: 2023年价格战后整车环节估值大幅分化，盈利能力的差异开始体现在估值上

## E — 可执行步骤 (Execution)

1. **确定估值框架**
   - 根据生命周期阶段选择估值方法（VC/PS/PEG/PE/PB）
   - 完成标准: 明确的估值方法选择 + 选择依据（基于生命周期阶段）

2. **收集估值数据**
   - **完整方法见 `reference/search-specs.md`「AkShare 本环境使用须知」**（本节为速查）
   - 数据标注参考: `reference/data-annotation-standards.md`
   - 完成标准: 主要上市公司的当前PE/PB + 行业均值 + 历史分位数（如有）

   **🔴 数据获取实操指引（本环境：Windows + Clash TUN 全局代理）**：

   ⚠️ **两个命门**（缺一不可，否则 AkShare 必失败）：① 须 `os.environ['NO_PROXY']='*'` 绕过 Windows 注册表系统代理；② **避开东财 push2 源**（`stock_zh_a_spot_em`/`stock_individual_info_em`/`stock_zh_a_hist` 等在 TUN 下不通），改用新浪/百度源。

   **方案一：AkShare（优先，可批量）**
   - 安装：`pip install akshare -q`
   ```python
   import os
   os.environ['NO_PROXY']='*'          # ← 命门①
   import akshare as ak
   # 行情（新浪源）：df = ak.stock_zh_a_spot()  # 代码格式 sh600900，筛选用 df[df['名称']=='长江电力']
   # 估值（百度源）：ak.stock_zh_valuation_baidu(symbol="600900", indicator="市盈率", period="近一年")
   #   indicator 可选：市盈率 / 市净率 / 总市值 / 股息率
   ```
   - 规律：数据源是新浪/百度/腾讯 → 可用；东财 push2 → 不可用

   **方案二：Chrome navigate 雪球个股页（单只取完整估值，或 AkShare 失败时备用）**
   - `browse(open, url="https://xueqiu.com/S/SH{code}", wait=networkidle)` → `extract(type="text")` 或 `evaluate` 提取含"市盈率/市净率/总市值/股息率"行
   - 雪球 SSR 完整估值表（PE-TTM/PB/市值/股息率），当日实时。**不用 WebFetch**（多数运行时不存在）、**不抓东财 push2**（TUN 下不通）

   **投资者版数据质量规则**：
   - 估值数据必须为**具体数值**（如"PE 24.9x"），不得使用区间（如"PE 15-25x"）
   - 每行数据标注获取来源（AkShare 新浪/百度源 / 雪球）和获取日期
   - 至少覆盖5家核心标的
   - 如 AkShare 与 Chrome 均失败 → 在估值章节用 🔴 显著标注"以下区间为 AI 估算，未获取实时数据"

   **其他版本（学生/求职者/创业者）**：至少覆盖3家核心标的，允许使用行业公开数据的合理区间，但需标注数据来源和时效性。

3. **估值判断**
   - 当前估值水平：偏低 / 合理 / 偏高
   - 有无比对标的海外成熟市场？估值差距如何？
   - 有无"预期差"？市场共识的偏差在哪里？
   - 完成标准: 给出合理的估值区间判断 + 核心依据

4. **估值驱动因素分析**
   - 识别影响该行业估值的关键变量（如渗透率、政策、技术突破）
   - 完成标准: 列出 3-5 个关键估值驱动因素

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 精确的DCF现金流折现估值——需要专业工具和完整财务数据，AI 无法完成
- 单只股票的买卖时机判断——本 skill 在行业层面（中观），不涉及个股交易
- 非上市公司的估值——缺乏公开市场数据

### 作者在书中警告的失败模式

- **市盈率滥用**：对亏损企业（导入期）算市盈率无意义；对强周期股（成熟期稳定周期化）用市盈率会买在利润高点
- **低估值陷阱**：衰退期PE很低不代表便宜——盈利可能在加速萎缩，PE会越跌越高
- **跨阶段用错估值方法**：导入期看PE、衰退期看增长——这是最常见的估值错误

### 作者的盲点 / 时代局限 (来自 BOOK_OVERVIEW.md)

- **A股估值体系中心**：不同资本市场的估值中枢差异显著（A股中小盘估值溢价 vs 美股机构主导的低估值）
- **理性预期假设**：估值框架隐含着"市场会逐渐发现价值"的假设，但作者自己也在情绪章节承认FOMO等非理性因素会主导短期估值
