# 假设驱动研究法 — 行业研究的元能力

## R — 原文 (Reading)

> 研究的本质是资讯的"输入 → 处理 → 输出"过程。资讯可以分为四个层级——数据(Data)、信息(Information)、知识(Knowledge)、智慧(Wisdom)。行业研究的目标是将"数据"升级为"知识"甚至"智慧"。
>
> 不是"先搜数据再填模板"，而是"先提假设再验证"——搜索的目的不是"发现数据"，而是"验证或推翻假设"。
>
> 麦肯锡分析师用金字塔原理完成资讯处理——"总结"与"提炼"有本质区别，缩句和归纳小学生都会，分析师的核心价值在于通过不断追问"那又怎样？"来产生深度洞见。
>
> — 肖璟, 第10-11章

## I — 方法论骨架 (Interpretation)

### 假设驱动研究法（核心工作流）

这是行业研究最重要的方法论——不是"先搜数据再填模板"，而是"先提假设再验证"：

```
步骤1: 提出初始假设
  "基于常识判断，我认为这个行业处于XX阶段，因为..."
  "假设这个行业的CR5大约在XX%..."
  "猜测这个行业的护城河主要在XX方面..."

步骤2: 定向搜索验证
  搜索的目的不是"发现数据"，而是"验证或推翻假设"
  → 找到支持证据 → 假设成立，继续深化
  → 找到反对证据 → 修正假设，重新搜索

步骤3: 修正并得出结论
  "最初假设CR5>70%，搜索后发现仅47%，属于低集中寡占型"
  "这意味着行业龙头有溢价空间但非垄断定价权"
```

### 金字塔原理：从信息中提炼核心观点

麦肯锡分析师用金字塔原理完成资讯处理。**"总结"与"提炼"有本质区别**——缩句和归纳小学生都会，分析师的核心价值在于通过不断追问"那又怎样？"（So what?）来产生深度洞见。

**构建金字塔的四步法**：

```
步骤1: 罗列资讯
  把所有重要发现写在"便签"上，用完整语句而非关键词

步骤2: 寻找规律
  提炼不同资讯的共性，用逻辑将相关事实串联分组
  常见分组维度：性质（What/Why/How）、影响（正/负）、时间（过去/现在/未来）

步骤3: 提炼要点
  为每个分组提炼一个要点（关键论述），回答"这一组资讯说明了什么？"
  ⚠️ 关键区别：提炼 ≠ 总结
    总结 = "A增长了20%，B下降了5%"  → 信息层级，无增量
    提炼 = "行业呈现结构性分化，龙头增速远超中小玩家" → 知识层级，有洞见

步骤4: 归纳核心
  思考各组要点对核心问题意味着什么 → 多问"那又怎样？"
  示例：
    "PMI连续4个月低于荣枯线" → So what? → "宏观经济不景气"
    "宏观经济不景气" → So what? → "可能需要出台稳增长政策" ← 这才是核心观点
```

**两种推理形式**：

| 推理类型 | 方式 | 适用场景 |
|---------|------|----------|
| **归纳推理** | 从具体事实中找出共性，提炼一般规律 | 数据充分、需要从现象中总结规律 |
| **演绎推理** | 从已知前提出发，通过逻辑推演得出结论 | 前提明确、需要推导因果关系 |

### SCQR 叙事框架：按受众调整报告结构

**"解题归解题，沟通归沟通"**——研究的推演顺序（提出假设→验证→结论）不等于向读者呈现的顺序。SCQR = 情景 → 冲突 → 问题 → 解答（又名"空雨湿伞"模型）：

| 要素 | 含义 | 示例（空雨湿伞） |
|------|------|----------------|
| **S** 情景 | 描述所处背景 | "天空飘来一朵云" |
| **C** 冲突 | 解释发生了什么变化 | "很可能要下雨" |
| **Q** 问题 | 带来了什么问题 | "可能会被淋湿而感冒" |
| **R** 解答 | 给出解决方案 | "带上一把伞" |

**按受众调整叙事顺序**：

| 受众类型 | 对应身份 | 推荐叙事顺序 | 理由 |
|---------|---------|------------|------|
| **路人** | 快速了解者 | C→Q→S→R（冲突前置） | 先吸引眼球，再给背景 |
| **观众** | 学生/研究者 | S→C→Q→R（经典SCQR） | 对方有兴趣，按逻辑展开 |
| **大老板** | 投资者/创业者 | R→S→C→Q（结论前置） | 时间宝贵，直接给结论 |
| **直属领导** | （不适用） | R only（仅给解答） | 已了解背景，只要方案 |

### DIKW 模型：理解研究的深度层级

资讯分为四个层级，深度逐级递增：

```
数据(Data) → 信息(Information) → 知识(Knowledge) → 智慧(Wisdom)
（原始材料）  （经处理具有逻辑关系）（经过滤提炼整合分析）  （能做出正确判断的方法论）
```

> 行业研究的目标是将"数据"升级为"知识"甚至"智慧"。仅仅列出市场规模数字是"数据"；与历史增速、渗透率关联分析才是"知识"。

### 三种信息差：研究如何创造价值

| 信息差 | 含义 | AI 的优势与劣势 | 应用策略 |
|--------|------|----------------|----------|
| **时间差** | 比别人更快获取关键信息 | ✅ AI 可并行抓取多源 | 多渠道同时搜索，不等待 |
| **广度差** | 比别人覆盖更多知识领域 | ✅ AI 最大优势——多学科视角 | 每个模块启用多学科视角 |
| **深度差** | 比别人理解得更透彻 | ⚠️ AI 弱项——缺失实地经验 | 诚实标注不确定性，引导人工深入 |

### 事实判断 vs 价值判断

| 类型 | 定义 | 特征 | 示例 |
|------|------|------|------|
| **事实判断** | 对客观存在的事物/关系的判断 | 可通过论据**证实或证伪** | "2025年渗透率30%" |
| **价值判断** | 对人/事/物的好坏及有用性的判断 | **因人而异**，无法证实 | "这个行业值得投资" |

> 高质量的价值判断必须有事实判断的支撑。报告中应明确区分两种判断——避免将价值判断伪装成事实判断。

## A1 — 书中的应用 (Past Application)

### 案例 1: 金字塔原理在工作中的应用
- **问题**: 研究得出的结论要如何呈现给决策者？
- **方法论的使用**: 某分析师写完长篇报告后，领导要求用金字塔原理重构——先给核心结论（塔尖），再展开支撑论据（塔身），最后是详细数据（塔基）。对于时间有限的"大老板"型受众，直接给塔尖即可。
- **结论**: 同样的研究内容，不同的呈现结构会产生完全不同的沟通效果

### 案例 2: 假设驱动法的高效验证
- **问题**: 面对一个新的行业，应该先全面搜集资料还是先建立假设？
- **方法论的使用**: 先基于常识建立 3-5 个核心假设（如"这个行业护城河主要在牌照"），然后定向搜索验证。找到反对证据时立即修正假设方向，而非继续寻找支持性证据。
- **结论**: 假设驱动法的时间效率远高于"先搜集再归纳"的填模板方式

### 案例 3: 专家访谈中的信息获取技巧
- **问题**: 如何从行业专家那里获得公开渠道无法获取的深度信息？
- **方法论的使用**: 绕开保密协议——不说"贵公司地推激励方案是什么？"改问"如果让您设计激励机制，会有哪些考虑？"；启发式追问——"这三家加起来市占率能有70%吗？"→ 专家修正为"80%"→ 追问"A比B和C高多少？"
- **结论**: 好的提问技巧能让专家在不违反保密协议的前提下提供关键判断依据

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 用户表示"信息太多了不知道怎么整理"，需要从海量搜索结果的"数据"提炼为"洞察"
2. 用户写出来的报告被反馈"只是数据堆砌、没有观点"，需要学习"提炼 vs 总结"的区别
3. 用户面对不同受众需要调整呈现方式——是给老板看（结论前置）还是给同行看（逻辑展开）
4. 用户在研究初期感到迷茫——不知道该从哪里开始搜索、先搜什么后搜什么
5. 用户对自己的分析信心不足，想确认"这个结论站得住脚吗"

### 语言信号

- "帮我分析一下XX" / "怎么做行业研究"
- "怎么从这些数据中提炼观点"
- "报告怎么写才有说服力"
- "So what?" / "那又怎样？"
- "金字塔原理" / "MECE"
- "这是事实还是观点？"

### 与相邻 skill 的区分

- 与 `spatial-temporal-benchmarking` 的区别: 假设驱动是**演绎验证**（先假设再证明），时空对标是**类比推理**（用过去/他处理解当下）。两者是不同的推理路径。
- 与 `industry-analysis-orchestrator` 的区别: 本 skill 是**元能力**——教你怎么做研究；orchestrator 是**总装线**——帮你执行研究。前者是方法，后者是执行。

## E — 可执行步骤 (Execution)

当 skill 被激活后，按以下步骤指导用户或自行执行：

1. **建立初始假设**
   - 基于已有认知，对核心问题提出 3-5 个可验证的假设
   - 每个假设必须是可被证实或证伪的（不是"这个行业不错"而是"这个行业渗透率约30%，处于成长期"）
   - 完成标准: 产出一份假设清单，每项标注"可验证的预测"

2. **定向搜索验证（而非全面搜集）**
   - 搜索的目的是验证假设，不是发现新事实
   - 同时搜索反面观点（如"{行业} 风险"、"为什么{行业}不行"）
   - 使用隐私/无痕模式搜索，减少个性化算法干扰
   - 完成标准: 每个假设有"支持证据"或"反对证据"的标注
   - 搜索执行参考: `reference/search-specs.md`
   - 数据标注参考: `reference/data-annotation-standards.md`

3. **应用金字塔原理提炼洞察**
   - 罗列所有重要发现→寻找规律→提炼要点→归纳核心
   - 对每个关键结论追问一次"那又怎样？"（So what?）
   - 区分"总结"（信息层级）和"提炼"（知识层级）
   - 完成标准: 每个关键分析点有一个"提炼"层级的结论

4. **选择叙事结构并输出**
   - 根据受众身份选择 SCQR 叙事顺序（见 I 节四类受众表）
   - 区分事实判断和价值判断——前者标注来源，后者说明依据
   - 完成标准: 报告开头/摘要的叙事顺序与受众匹配

5. **自检清单**
   - 参考 `reference/data-annotation-standards.md` 中的报告质量自检清单
   - 检查是否应用了 MECE 原则（分类无重叠、无遗漏）

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 用户只需要一个具体的数据或事实（如"2025年新能源汽车渗透率是多少"）——这是信息检索，不是研究
- 用户已经有了明确的结论，只是想美化报告格式——本 skill 关注内容深度而非形式美化
- 用户处于极度时间紧迫的状态（如"5分钟内给我答案"）——假设驱动法需要多轮迭代

### 作者在书中警告的失败模式

- **FOMO（错失恐惧症）导致广度牺牲深度**: 在未建立深度差之前过度追求覆盖更多行业/话题，容易浅尝辄止。宁可把核心模块做深做透，也不要为了"覆盖所有模块"而每个都蜻蜓点水。
- **系统1直觉替代系统2分析**: 容易凭直觉快速下结论（系统1），但行业研究要求用数据和逻辑验证直觉（系统2）。特别是在看到支持自己已有观点的数据时，更容易跳过验证步骤。
- **信息茧房**: 个性化推荐算法强化了人们只看自己认同信息的倾向。必须主动搜索反面观点，用无痕模式减少算法干扰。
- **价值判断伪装成事实判断**: "这个行业前景很好"是价值判断，需要有"市场规模年增30%+渗透率仅15%+政策大力支持"等事实判断的支撑。

### 作者的盲点 / 时代局限 (来自 BOOK_OVERVIEW.md)

- 本书的假设驱动法默认研究者有一定行业知识积累（麦肯锡分析师通常有 MBA 背景），对于完全零基础的初学者，最初 1-2 轮假设可能偏差极大——需要更多迭代
- 金字塔原理要求"相互独立、完全穷尽"（MECE），但现实中很多行业问题存在交叉和灰色地带，过度追求 MECE 可能导致强行分类

### 容易混淆的邻近方法论

- **波特五力 vs 金字塔原理**: 波特五力是**分析框架**（What to analyze），金字塔原理是**输出结构**（How to present）。两者不冲突——可以用波特五力做分析，用金字塔原理做呈现。
- **SWOT分析 vs 假设驱动**: SWOT是分类工具（把信息放进四象限），假设驱动是研究路径（先猜再验证）。假设驱动效率更高。
